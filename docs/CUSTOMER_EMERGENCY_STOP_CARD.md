# お客様向け緊急停止カード

更新日：2026-07-18（JST）  
対象：Threads自動投稿環境 初期モニター

## このカードを使う場面

次のいずれかが起きた場合は、通常操作を続けず、このカードに従ってください。

- 投稿されたか分からない
- 同じ投稿が重複した可能性がある
- 意図しない投稿が行われた
- GitHub Actionsが失敗した
- ツリー投稿が途中で止まった
- Threadsでアカウント制限、警告、審査表示が出た
- 投稿履歴とThreads実画面が一致しない

## まず止める

1. Workflowを手動実行しない
2. Actionsの再実行ボタンを押さない
3. `error`を`ready`へ戻さない
4. 投稿予定を削除しない
5. 投稿済みデータ、投稿ID、投稿日時を変更しない
6. 同じ内容を新規投稿として登録し直さない
7. パスワード、トークン、Secrets、二段階認証コードを送らない

投稿済みか不明な状態で再実行すると、二重投稿になる可能性があります。

## 利用者が確認すること

技術的な調査は行わず、次だけ確認してください。

- Threads実画面に対象投稿があるか
- 対象投稿の予定日時
- 表示されているエラーまたは警告の文章
- 直前に行った操作
- 手動実行や再実行を行ったか

認証画面、Secrets画面、トークンが表示される画面のスクリーンショットは送らないでください。

## アカウント制限・警告・審査が表示された場合

- 新しい投稿予約を`ready`にしない
- 手動投稿を含め、追加操作を続けない
- 自動投稿の安全ロックが必要かgoroへ連絡する
- 制限解除を推測して再開しない

安全ロックの解除は、Threads実画面の状態とgoroの確認後に行います。

## goroへ送る連絡文

分かる範囲で次を埋めてください。不明な項目は「不明」とします。

```text
【緊急サポート依頼】

発生日時：
困っていること：
対象投稿の予定日時：
Threads実画面：投稿あり / 投稿なし / 不明
表示されたエラーまたは警告：
直前に行った操作：
手動実行・再実行：した / していない / 不明
対象リポジトリURL：
コラボレーター権限：維持中 / 削除済み / 不明
```

次の情報は含めないでください。

- パスワード
- Threadsアクセストークン
- GitHubトークン
- GitHub Secretsの値
- 二段階認証コード
- バックアップコード
- 不要な個人情報

## 連絡後

goroから案内があるまで、対象投稿とWorkflowを変更しないでください。

Threads実画面に投稿がある場合も、履歴書き戻しが失敗している可能性があるため、自己判断で同じ投稿を再送しないでください。
