# お客様向けクイックスタート

更新日：2026-07-18（JST）  
対象：Threads自動投稿環境 初期モニター

## この資料の目的

日常利用で最低限必要な操作と、安全のために守ることだけをまとめます。

詳しい操作や技術調査は、購入者向けサポートGPTまたはgoroへ相談してください。利用者自身でPython、GitHub Actions、Workflow、cronを変更する必要はありません。

## 普段使う画面

日常の投稿管理は、導入時に決めた次のいずれかを使用します。

- ChatGPT通常チャット＋GitHub連携
- Claude通常チャット＋公式GitHub MCP

GitHubは投稿予定、投稿履歴、設定、実行結果を保存する場所です。通常操作ではGitHubのファイルを直接編集しません。

## 投稿の基本手順

1. チャットへ投稿本文と希望日時を伝える
2. 最初は`draft`で登録する
3. 本文、日時、画像、類似投稿を確認する
4. 確定した投稿だけ`ready`にする
5. 投稿後に`posted`になったことを確認する

## 投稿状態

| 状態 | 意味 |
|---|---|
| `draft` | 下書き。自動投稿されない |
| `ready` | 自動投稿の対象 |
| `posting` | 投稿処理中 |
| `posted` | 投稿済み |
| `error` | 失敗または確認が必要。再実行しない |
| `cancelled` | キャンセル済み |

## そのまま使える依頼例

### 投稿を下書き登録

```text
2026年7月20日の20時に、次の文章をThreads投稿予定として追加してください。
過去の投稿との重複を確認し、まずdraftで保存してください。

（投稿本文）
```

### 内容確認後に自動投稿対象へ変更

```text
対象投稿の本文、日時、画像、投稿先を確認しました。
同じ時刻のready投稿と類似投稿をもう一度確認し、問題がなければreadyにしてください。
変更した投稿ID、日時、statusを報告してください。
```

### 投稿予定を確認

```text
今日から7日間のThreads投稿予定を、日時、タイトル、投稿ID、status付きで一覧にしてください。
ファイルは変更しないでください。
```

### キャンセル

```text
対象の投稿予定を削除せず、statusをcancelledにしてください。
変更後の投稿ID、日時、statusを報告してください。
```

## 必ず守ること

- パスワード、アクセストークン、GitHubトークン、Secrets、二段階認証コードをチャット、LINE、メールへ送らない
- `ready`にすると自動投稿対象になる
- 投稿時刻まで30分以内、または予定時刻を過ぎている投稿は、内容を確認せず`ready`にしない
- 投稿済みデータ、投稿ID、投稿日時を推測で変更しない
- キャンセル時は削除せず`cancelled`にする
- 投稿済みか不明な場合は、手動実行や再実行をしない
- アカウント制限、警告、審査表示が出た場合は操作を止める

## 投稿時刻について

本サービスでは任意の投稿日時を指定できますが、GitHub Actionsの性質上、指定時刻ぴったりの投稿は保証しません。遅延や実行欠落が起こる可能性があります。

情報解禁、販売開始、イベント開始など、時刻を絶対にずらせない投稿はThreads公式の予約投稿を使用してください。

詳しくは`CUSTOMER_POSTING_TIME_GUIDE.md`を確認してください。

## 困ったとき

次の状況では操作を止め、購入者向けサポートGPTまたはgoroへ連絡してください。

- 投稿されたか分からない
- 同じ投稿が重複した可能性がある
- GitHub Actionsが失敗した
- Threadsでアカウント制限、警告、審査が表示された
- ツリー投稿が途中で止まった
- 意図しない投稿が行われた

緊急時は`CUSTOMER_EMERGENCY_STOP_CARD.md`に従ってください。

## 連絡先の使い分け

- 日常の相談、日程調整：LINE公式
- 契約、返金、停止、サポート終了などの重要連絡：メール
- 基本操作、一次確認：購入者向けサポートGPT
- 技術調査、復旧、設定変更：goro
