# v0.1 販売前テスト計画

## 1. 静的検証

- [ ] `Validate Template` workflowが成功する
- [ ] Python構文エラーがない
- [ ] `config/service.yml`を読み込める
- [ ] 未定義の投稿枠を検出できる
- [ ] `ready`なのに画像URLがない投稿を検出できる
- [ ] 重複した投稿IDを検出できる

## 2. テキスト投稿

- [ ] プロフィール取得だけの接続テストが成功する
- [ ] 短いテキストを手動テスト投稿できる
- [ ] dry runでは投稿されない
- [ ] 自動投稿後に`status: posted`へ変わる
- [ ] `threads_post_id`と`posted_at`が記録される
- [ ] `posts/posted_log.yml`へ履歴が追加される
- [ ] 同じworkflowを再実行しても二重投稿されない

## 3. 時刻判定

- [ ] `config/service.yml`の投稿枠を読み取る
- [ ] 投稿枠より前には投稿されない
- [ ] 投稿枠の確認幅内で投稿される
- [ ] 確認幅を過ぎた投稿は自動実行されない
- [ ] 23時台の遅延を日付変更後に回収できる
- [ ] 投稿枠を追加してもworkflowのcron変更が不要

## 4. 画像投稿

- [ ] Public画像リポジトリを作成する
- [ ] Variablesを設定する
- [ ] 画像用Fine-grained PATをSecretsへ登録する
- [ ] PNGをWebPへ変換できる
- [ ] JPGをWebPへ変換できる
- [ ] 透過PNGをWebPへ変換できる
- [ ] Public画像リポジトリへ保存できる
- [ ] 公開Raw URLへアクセスできる
- [ ] `image_key`が一致する投稿へURLが反映される
- [ ] URL反映後に`draft`から`ready`へ変わる
- [ ] 画像付き投稿がThreadsへ表示される
- [ ] 画像処理後に元画像が投入場所から削除される

## 5. ツリー投稿

- [ ] 2件のツリー投稿が成功する
- [ ] 3件のツリー投稿が成功する
- [ ] 各返信IDが記録される
- [ ] 親子関係が正しい
- [ ] 返信に画像を付けられる
- [ ] 親投稿失敗時に返信処理が始まらない
- [ ] 途中失敗時に`status: error`になる
- [ ] 途中失敗後の安全な復旧手順を決める

## 6. エラー対応

- [ ] Threads Secret未登録時の表示を確認する
- [ ] 無効なアクセストークン時の表示を確認する
- [ ] 画像用トークン未登録時の表示を確認する
- [ ] 画像リポジトリ名が間違っている場合を確認する
- [ ] 失敗時に機密値がログへ表示されない
- [ ] 投稿失敗時に原因がYAMLへ記録される
- [ ] 画像処理失敗時にIssueが作成される
- [ ] 投稿処理失敗時にIssueが作成される

## 7. 顧客環境リハーサル

- [ ] 新規テスト用GitHubアカウント相当の環境を用意する
- [ ] テンプレートからPrivateリポジトリを作成する
- [ ] 画像用Publicリポジトリを作成する
- [ ] 顧客本人だけでSecretsを登録できる
- [ ] 初期設定開始からテスト投稿までの時間を測る
- [ ] 説明が必要だった箇所を記録する
- [ ] ChatGPT／Codexから投稿を追加できる
- [ ] ChatGPTへ投稿予定を質問できる
- [ ] ChatGPTへ投稿履歴を質問できる
- [ ] 類似投稿を検索できる
- [ ] 契約終了時の権限削除をリハーサルする

## 合格条件

次を満たすまで、有料モニター向け完成品として扱わない。

- テキスト・画像・ツリー投稿が再現可能
- 二重投稿防止を確認済み
- 機密情報を運営者が受け取らずに導入可能
- エラー時に原因を特定できる
- 顧客環境への導入時間を把握済み
- 途中失敗後の復旧手順を文書化済み
