- workflow：`Test Thread Failure Recovery`
- 実行方式：モックAPI
- Threads APIとGitHub Secretsは不使用
- workflow権限：`contents: read`
- 親投稿失敗時はAPI呼び出し1回で終了し、返信処理なし
- 途中失敗時は成功済み親・返信IDと`completed_replies`を保持
- `error`状態は自動候補にならない
- 明示的に`posting`へ戻した場合のみ未完了返信から再開
- 完了後の再実行でAPI呼び出し・履歴重複なし
- 顧客相当環境と汎用テンプレートの両方でworkflow成功を確認

- エラー時に原因を特定できる
- 顧客環境への導入時間を把握済み
- 途中失敗後の復旧手順を文書化済み
- 投稿履歴検索・類似投稿検索を検証済み
- ChatGPT／Codexを標準操作環境として検証済み
- 通常のClaudeチャットを対応環境として表記する場合は、GitHub MCP互換性項目を検証済み
- Claude Codeを対応環境として表記する場合は、Claude Code互換性項目を検証済み
