# ChatGPT／Codexでの管理例

## 投稿予定を追加する

```text
2026年7月5日の朝7時に、次の文章をThreadsへ投稿予定として追加してください。
過去の投稿と重複していないか確認してから、投稿可能な状態にしてください。

（ここに本文）
```

確認が済んでいない場合は、`status: draft`で保存します。

## 下書きとして追加する

```text
次の文章を、日付未確定の投稿案として整理してください。
自動投稿可能な状態にはしないでください。
```

日付未確定の案は、実際の週別投稿ファイルとは別の下書きファイルへ保存する運用も検討します。

## 投稿予定を確認する

```text
今日から7日間の投稿予定を、日時・タイトル・状態付きで一覧にしてください。
ファイルは変更しないでください。
```

## 投稿履歴を確認する

```text
先月投稿した内容を、日付・投稿枠・タイトル・投稿ID付きで一覧にしてください。
```

## 類似投稿を探す

```text
次の文章と似た過去投稿があるか確認してください。
候補を示すだけで、投稿データは変更しないでください。

（ここに本文）
```

## 日時を変更する

```text
IDが post_20260705_001 の未投稿データを、2026年7月6日の20時へ移動してください。
移動先に別の投稿がないか確認してください。
```

## キャンセルする

```text
IDが post_20260705_001 の投稿予定をキャンセルしてください。
履歴確認のため削除はせず、cancelledに変更してください。
```

## 画像付き投稿を追加する

```text
2026年7月7日の20時に、次の文章と画像を使う投稿予定を追加してください。
画像ファイル名は campaign_20260707.png です。
画像URLが反映されるまではdraftにしてください。

（ここに本文）
```

YAMLでは拡張子を除いた`campaign_20260707`を`image_key`にします。

## ツリー投稿を追加する

```text
次の3つの文章を、上から順にThreadsのツリー投稿として登録してください。
1件目をルート、2件目と3件目を返信にしてください。

1件目：...
2件目：...
3件目：...
```

## 変更内容を確認する

ChatGPT／Codexへ編集を依頼した後は、次を確認します。

- 対象日と投稿時刻
- 本文
- `status`
- 画像キーと画像URL
- ツリーの順序
- 既存投稿が変更されていないか
- 投稿済み履歴が消えていないか

重要な投稿は、コミット差分を確認してから`ready`にします。
