# 通常のClaudeチャットでの投稿管理

この文書は、通常のClaudeチャットへ公式GitHub MCPコネクタを接続し、対象リポジトリへ必要な読み書き権限を付与して利用する場合の手順です。

## 前提

- 対象リポジトリへのアクセス権がある
- GitHub MCPコネクタがClaudeチャットへ接続されている
- YAMLを編集する場合は、対象リポジトリへの書き込み権限がある
- 読み取り専用設定ではファイル作成・更新はできない
- Secrets、トークン、パスワードをClaudeへ渡さない

## 最初に依頼すること

```text
このGitHubリポジトリのCLAUDE.mdとAGENTS.mdを読み、記載された安全ルールに従ってください。
通常の投稿管理ではposts/とconfig/service.yml以外を変更しないでください。
Secrets、トークン、パスワードは検索・表示・要求しないでください。
```

## 投稿予定を追加する

```text
明日の朝7時に、次の文章をThreads投稿予定へ追加してください。
過去投稿との重複を確認してください。
投稿時刻まで30分以内、確認事項がある、または類似投稿がある場合はreadyにせずdraftにしてください。

（ここに本文）
```

## 投稿予定を確認する

```text
今日から7日間のThreads投稿予定を、日時、タイトル、status、投稿IDの有無付きで一覧にしてください。
ファイルは変更しないでください。
```

## 日時を変更する

```text
IDがpost_20260705_001の未投稿データを、2026年7月6日のeveningへ移動してください。
移動先に別のready投稿がないか確認し、IDと本文は変更しないでください。
```

## キャンセルする

```text
IDがpost_20260705_001の予定を削除せず、statusをcancelledへ変更してください。
必要に応じてcancelled_atとcancel_reasonも記録してください。
```

## 類似投稿を探す

```text
次の文章と似た投稿が過去にないか確認してください。
候補を示すだけで、ファイルは変更しないでください。

（ここに本文）
```

## 画像付き投稿

```text
2026年7月7日のeveningに、次の文章とcampaign_20260707.pngを使う投稿予定を追加してください。
image_keyは拡張子を除いた値にしてください。
image_urlが反映されるまではdraftにしてください。

（ここに本文）
```

## GitHub Actionsを確認する

```text
このリポジトリの直近のGitHub Actions実行結果を確認してください。
失敗している場合は、ログから原因候補を説明してください。
Secretsの値は表示しないでください。
修正はまだ行わないでください。
```

## 作業後の報告

- 変更したファイル
- 追加・変更した投稿ID
- 投稿日時と投稿枠
- 最終status
- 重複確認結果
- 投稿済みデータを変更していないこと
- `src/`や`.github/workflows/`を変更していないこと

## 正式対応の検証項目

- 投稿予定の追加
- 日時変更
- キャンセル
- 履歴検索
- 類似投稿検索
- 画像付き投稿のYAML作成
- ツリー投稿のYAML作成
- Issue・Pull Request操作
- GitHub Actionsの確認
- `posted`データの保護
- Secretsを要求・表示しないこと
- 意図しないコード変更を行わないこと
