# お客様向け導入完了確認チェックリスト

更新日：2026-07-18（JST）  
対象：Threads自動投稿環境 初期モニター

## 目的

初期設定の終了時に、利用者とgoroが一緒に確認します。

このチェックリストの完了を、サポート期間開始と日常運用へ移るための確認記録として使用します。

## 1. 利用環境

- [ ] 利用するAIチャットを確認した
  - [ ] ChatGPT通常チャット＋GitHub連携
  - [ ] Claude通常チャット＋公式GitHub MCP
- [ ] 顧客所有のPrivate自動投稿リポジトリを確認した
- [ ] 画像投稿を利用する場合、顧客所有のPublic画像リポジトリを確認した
- [ ] goroのGitHubコラボレーター権限がサポート期間中維持されることを確認した

## 2. 機密情報

- [ ] パスワード、アクセストークン、GitHubトークン、Secrets、二段階認証コードをgoroやAIへ送らないことを確認した
- [ ] Secretsは利用者本人がGitHubの設定画面へ入力した
- [ ] 認証情報が表示される画面では、必要に応じて画面共有を停止した
- [ ] 画像用トークンを使用する場合、対象画像リポジトリだけに必要最小限の権限を設定した

## 3. 接続と投稿テスト

- [ ] Threads接続確認に成功した
- [ ] dry runで対象投稿と本文を確認した
- [ ] 少数のテスト投稿に成功した
- [ ] 投稿後に`status: posted`、投稿ID、投稿日時が記録されたことを確認した
- [ ] 画像投稿を利用する場合、画像がThreads上で正しく表示された
- [ ] ツリー投稿を利用する場合、親子関係と表示順を確認した

## 4. 日常操作

- [ ] 新しい投稿は原則`draft`で登録することを確認した
- [ ] `ready`にすると自動投稿対象になることを確認した
- [ ] 予定時刻まで30分以内、または時刻経過後は、確認なしに`ready`にしないことを確認した
- [ ] キャンセル時は削除せず`cancelled`にすることを確認した
- [ ] `posted`、投稿ID、投稿日時を推測で変更しないことを確認した
- [ ] 投稿予定と投稿履歴の確認方法を実際に試した
- [ ] 購入者向けサポートGPTの利用方法を確認した

## 5. 投稿時刻

- [ ] 初期設定する標準投稿枠を確認した

| 枠名 | 時刻（JST） | 利用する |
|---|---|---|
| morning |  | [ ] |
| noon |  | [ ] |
| afternoon |  | [ ] |
| evening |  | [ ] |
| night |  | [ ] |
| その他 |  | [ ] |

- [ ] チャットから任意日時を指定できることを確認した
- [ ] 任意日時の指定は、時刻ぴったりの投稿保証ではないことを確認した
- [ ] GitHub Actionsには遅延や実行欠落の可能性があることを確認した
- [ ] 情報解禁、販売開始、イベント開始など時刻厳守の投稿には、Threads公式の予約投稿を使用することを確認した

## 6. 画像公開

画像投稿を利用する場合に確認します。

- [ ] Public画像リポジトリへ置いた画像は、投稿前でもインターネット上から閲覧可能になることを確認した
- [ ] 個人情報、機密情報、事前公開できない画像を登録しないことを確認した
- [ ] 使用する画像の利用権利を利用者が確認することを確認した
- [ ] 大量登録、画像生成、複雑な画像加工は標準サポート外または個別相談になることを確認した

## 7. 緊急時

- [ ] 投稿済みか不明な場合は再実行しないことを確認した
- [ ] 二重投稿、ツリー途中停止、履歴不一致では操作を止めることを確認した
- [ ] Threadsで制限、警告、審査表示が出た場合は追加操作を止めることを確認した
- [ ] `CUSTOMER_EMERGENCY_STOP_CARD.md`の内容を確認した
- [ ] 安全ロックの位置づけと、解除を自己判断しないことを確認した

## 8. サービス範囲

- [ ] サポート期間の開始日と終了日を確認した
- [ ] 日常相談はLINE公式、重要連絡はメールを使用することを確認した
- [ ] 投稿時刻、売上、フォロワー増加、アカウント制限回避を保証するサービスではないことを確認した
- [ ] Threads以外のSNS、無制限の投稿文制作、画像制作、動画制作は標準範囲外であることを確認した
- [ ] 追加作業や対象外作業は、実施前に内容と料金を確認することを確認した

## 完了判定

次の項目が未確認の場合は、導入完了扱いにせず、goroと利用者で確認を続けます。

- Threads接続
- テスト投稿
- Secretsを送らないルール
- `draft`と`ready`の違い
- 投稿時刻の非保証と公式予約の使い分け
- 緊急時の再実行禁止
- 連絡窓口
- サポート期間
