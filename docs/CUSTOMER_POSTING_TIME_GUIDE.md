# お客様向け投稿時刻の使い分けガイド

更新日：2026-07-18（JST）  
対象：Threads自動投稿環境 初期モニター

## 結論

本サービスでは、初期導入時に設定した標準投稿枠と、チャットから指定する任意日時を利用できます。

ただし、任意の日時を指定できることと、指定時刻ぴったりに投稿されることは別です。

GitHub ActionsのScheduled実行には遅延や実行欠落の可能性があるため、時刻を絶対にずらせない投稿はThreads公式の予約投稿を使用してください。

## 使い分け

| 投稿の種類 | 推奨方法 |
|---|---|
| 普段の朝・昼・夜の投稿 | 初期設定済みの標準投稿枠 |
| 数分から十数分のずれが許容される投稿 | 本サービスの自動投稿 |
| 通常と異なる日時に出したい投稿 | チャットから任意日時を指定 |
| 情報解禁時刻が決まっている投稿 | Threads公式の予約投稿 |
| セール開始・終了時刻と連動する投稿 | Threads公式の予約投稿 |
| イベント、ライブ開始時刻に合わせる投稿 | Threads公式の予約投稿 |
| 契約や取引先との約束で時刻厳守の投稿 | Threads公式の予約投稿 |

## 標準投稿枠

導入時に、利用者が普段使う時刻を確認して設定します。

例：

- morning：07:00
- noon：12:00
- night：20:00

標準投稿枠は日常利用を簡単にするためのものです。利用者ごとの希望に合わせて初期設定し、必要に応じてgoroが見直します。

## 任意日時の予約

チャットでは、標準投稿枠以外の具体的な日時も指定できます。

依頼例：

```text
2026年7月25日の13時40分に、次の文章をThreads投稿予定として追加してください。
この時刻で自動投稿可能か確認し、まずdraftで保存してください。
指定時刻ぴったりの保証がないことも確認してください。

（投稿本文）
```

任意日時の予約機能は、時刻厳守を保証する機能ではありません。

## 時刻厳守か判断する目安

次の言葉や条件がある場合は、本サービスでの自動予約だけを勧めず、Threads公式予約の利用を案内します。

- 「必ず」「絶対」「ちょうど」「厳守」
- 「情報解禁」
- 「販売開始」「受付開始」「締切」
- 「イベント開始」「ライブ開始」
- 数分の遅れでも問題が発生する
- 取引先、契約、キャンペーン条件で日時が決められている

確認文：

```text
この投稿は、指定時刻から数分以上遅れると問題がありますか？
```

問題がある場合の案内：

```text
この投稿は時刻厳守のため、本サービスの自動投稿ではなく、Threads公式の予約投稿をご利用ください。
本サービスの予約は指定日時を登録できますが、GitHub Actionsの遅延や実行欠落により、時刻ぴったりの投稿は保証できません。
```

## 遅延した場合

予定時刻を過ぎても投稿されない場合、利用者自身でWorkflowを再実行しないでください。

確認するのは次の3点だけです。

1. Threads実画面に投稿があるか
2. 指定した投稿予定日時に間違いがないか
3. 画面にエラーや警告が表示されているか

投稿済みか不明、エラー表示あり、十分に時間が経過しても未投稿の場合はgoroへ連絡してください。
