# Claude Codeでの投稿管理

この文書は、Claude Codeが対象リポジトリへ読み書きできる状態で利用することを前提とします。

## 最初に依頼すること

```text
このリポジトリのCLAUDE.mdとAGENTS.mdを読み、ルールに従って作業してください。
通常の投稿管理ではposts/とconfig/service.yml以外を変更しないでください。
```

## 投稿予定を追加する

```text
明日の朝7時に、次の文章をThreadsの投稿予定へ追加してください。
過去投稿との重複を確認してください。
投稿時刻まで30分以内、または確認事項がある場合はreadyにせずdraftにしてください。

（ここに本文）
```

## 投稿予定を確認する

```text
今日から7日間のThreads投稿予定を、日時、タイトル、状態、投稿IDの有無付きで一覧にしてください。
ファイルは変更しないでください。
```

## 投稿履歴を確認する

```text
先月のThreads投稿履歴を、日付、投稿枠、タイトル、投稿ID付きでまとめてください。
posts/schedulesとposts/posted_log.ymlの両方を確認してください。
```

## 類似投稿を探す

```text
次の文章と似た投稿が過去にないか確認してください。
候補を示すだけで、ファイルは変更しないでください。

（ここに本文）
```

## 日時を変更する

```text
IDがpost_20260705_001の未投稿データを、2026年7月6日のeveningへ移動してください。
移動先に別のready投稿がないか確認し、変更後の差分を説明してください。
```

## キャンセルする

```text
IDがpost_20260705_001の予定を削除せず、cancelledへ変更してください。
キャンセル日時と理由も記録してください。
```

## 画像付き投稿

```text
2026年7月7日のeveningに、次の文章とcampaign_20260707.pngを使う投稿予定を追加してください。
image_keyは拡張子を除いた値にしてください。
画像URLが反映されるまではdraftにしてください。

（ここに本文）
```

## 作業後に確認すること

- 変更したファイル
- 追加・変更した投稿ID
- 投稿日時と投稿枠
- `status`
- 重複確認結果
- 画像URLの有無
- 投稿済み履歴を変更していないこと
- `src/`や`.github/workflows/`を変更していないこと

## 正式対応の条件

次の検証が完了するまでは、Claude Code対応は試験運用として扱います。

- 投稿予定の追加
- 日時変更
- キャンセル
- 履歴検索
- 類似投稿検索
- 画像付き投稿のYAML作成
- ツリー投稿のYAML作成
- `posted`データを保護できること
- Secretsを要求・表示しないこと
- 意図しないコード変更を行わないこと
